# Detection and Response Lab

### Objective

During the Detection and Response Lab, I gained hands-on experience identifying and responding to cybersecurity threats through practical, scenario-based exercises. I learned to use industry-standard tools such as SIEMs, intrusion detection systems (IDS/IPS), and log analysis platforms to detect suspicious activities across networks and endpoints.  

This lab helped me understand how to simulate cyber-attacks and monitor system behavior, providing practical insight into how real-world threats unfold. I developed skills in analyzing alerts, correlating events from multiple sources, and conducting threat hunting using log data. Most importantly, I learned to respond to incidents by applying containment, eradication, and recovery strategies, as well as documenting and reporting findings clearly and professionally.  

This experience strengthened my confidence in threat detection, incident response, and making decisive actions during potential security breaches.

### Skills Learned

- Detected suspicious activities by analyzing security logs from endpoints and network devices.  
- Used tools like **Splunk**, **Suricata**, and log forwarders to monitor and investigate threats.  
- Simulated attacks such as phishing, brute-force, and privilege escalation to understand attacker behavior.  
- Conducted proactive threat hunting using log data to identify potential risks.  
- Applied key incident response strategies: containment, eradication, and recovery.  
- Improved documentation skills by recording investigation steps and summarizing findings clearly.  
- Strengthened critical thinking and decision-making skills in high-pressure scenarios.  
- Increased confidence in detecting, analyzing, and responding to cybersecurity threats.

### Tools Used

- **Splunk** – Log aggregation, searching, and analyzing security events.  
- **Suricata** – IDS/IPS deployed to monitor network traffic.  
- **Log Forwarder** – Forwarded logs from endpoints to analysis tools like Splunk and Chronicle.  
- **Wireshark** – Analyzed packet captures and network traffic during simulated attacks.
